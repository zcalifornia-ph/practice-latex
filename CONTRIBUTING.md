# Contributing

Thanks for contributing to `practice-latex`.
This repository is used to learn LaTeX fundamentals, packages, and typesetting workflows through practical examples and notes.

## Before You Start

- Check open issues and pull requests to avoid overlap.
- Open an issue first for major structural changes.
- Keep changes scoped and easy to review.
- Follow `CODE_OF_CONDUCT.md` and `SECURITY.md`.

## What to Contribute

- LaTeX examples (`.tex`) that teach one concept clearly.
- Notes or references that improve learning flow.
- Fixes for formatting, grammar, broken links, or incorrect commands.
- Better organization for docs, examples, and learning paths.

## Contribution Standards

- Make examples reproducible.
- Keep explanations concise and accurate.
- Cite external references when used.
- Prefer maintainable structure over one-off hacks.

## LaTeX Quality Checklist

- Source files compile without blocking errors.
- Warning-heavy output is documented if intentional.
- Figures/tables/references are linked correctly.
- File and folder names are descriptive and consistent.

## Workflow

1. Fork the repository.
2. Create a branch from `main`.
3. Make and verify your changes locally.
4. Open a pull request with:
   - what changed
   - why it changed
   - how you validated it

## Branch Naming

Use concise kebab-case with a Conventional Commit prefix.

```text
docs/add-basic-equations-module
feat/add-bibliography-examples
```

## Commit Format

Use Conventional Commits:

```text
docs: add LaTeX sectioning examples
feat: add bibliography and citation sample
fix: correct broken image path in README
```

## Pull Request Notes

- Keep each pull request focused on one logical topic.
- Link related issues when applicable.
- Include screenshots or rendered PDF snippets for visual changes when useful.

## Security Reporting

Do not open public issues for vulnerabilities.
Report security concerns through `SECURITY.md`.
